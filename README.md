# Smart Lender — Loan Approval Predictor

A machine-learning-powered Flask web app that predicts loan approval
likelihood from applicant details, using Decision Tree, Random Forest, KNN,
and XGBoost classifiers (the best-performing model is auto-selected and
served).

## Project structure
```
smart-lender-app/
├── data/
│   ├── generate_data.py   # creates a synthetic loan_data.csv (swap in your real dataset)
│   └── loan_data.csv
├── model/
│   ├── model.pkl              # saved best model + encoders (created by train.py)
│   └── confusion_matrix.png   # evaluation chart (created by train.py)
├── templates/
│   └── index.html         # prediction form UI
├── train.py                # trains all 4 models, picks the best, saves it
├── app.py                  # Flask web app that serves predictions
└── requirements.txt
```

## 1. Install dependencies
```bash
cd smart-lender-app
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

## 2. (Optional) Use your own dataset
The included `data/loan_data.csv` is **synthetic** — generated by
`data/generate_data.py` — so the app works out of the box. To use real data,
replace `data/loan_data.csv` with your own CSV, keeping these column names:

```
Gender, Married, Dependents, Education, Self_Employed,
ApplicantIncome, CoapplicantIncome, LoanAmount, Loan_Amount_Term,
Credit_History, Property_Area, Loan_Status
```

## 3. Train the models
```bash
python train.py
```
This prints train/test accuracy for all four models, picks the best by test
accuracy, and saves it to `model/model.pkl` along with a confusion-matrix
chart at `model/confusion_matrix.png`.

## 4. Run the web app
```bash
python app.py
```
Open **http://localhost:5000** — fill in the form and click "Predict
approval" to get an instant result.

## Notes
- If `xgboost` fails to install on your machine (e.g. missing OpenMP on
  macOS), `train.py` will automatically skip it and still train/compare the
  other three models — just run `brew install libomp` (Mac) or use a
  pre-built wheel and re-run.
- To deploy this (e.g. to IBM Cloud, Render, or Railway), make sure
  `model/model.pkl` is committed/available at deploy time, or that `train.py`
  runs once as part of your build step.
